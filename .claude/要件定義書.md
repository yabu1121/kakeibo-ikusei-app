## 1. アプリケーション概要

### 1-1. プロダクト名
「家計の守護獣（仮称）」

### 1-2. プロダクトビジョン
家計簿の継続を「面倒な義務」から「楽しい習慣」に変える。支出記録をキャラクター育成と紐付けることで、ユーザーに「自分の生活を自分でコントロールしている手応え」を提供する。

### 1-3. ターゲットユーザー
- クレジットカード連携に抵抗があり、手入力で家計管理したい人
- 家計簿アプリを始めても三日坊主になりがちな人
- ゲーム的要素でモチベーションを維持したい人
- 年齢層：20代〜40代
- デバイス：スマートフォンメイン

### 1-4. コア価値
- **手入力へのこだわり**：自分で記録することで支出への意識を高める
- **癒やし**：キャラクターの存在が記録継続のモチベーションになる
- **プライバシー第一**：銀行・クレカ連携なし、データは自分で管理

## 2. 機能要件

### 2-1. 認証・アカウント管理

#### 2-1-1. アカウント登録
- **登録方法**：メールアドレス＋パスワード
- **必須項目**：
  - メールアドレス（重複不可、形式バリデーション）
  - パスワード（8文字以上、英数字混在必須）
- **任意項目**：なし
- **メール確認**：
  - 登録時に確認メール送信
  - メール内のリンクをクリックで認証完了
  - 未確認の場合、ログインは可能だが一部機能制限（データエクスポート不可など）
  - 確認メールの再送機能あり
- **SNSログイン**：初期実装なし（将来拡張）

#### 2-1-2. ログイン
- **ログイン方法**：メールアドレス＋パスワード
- **セッション管理**：JWT使用、有効期限7日間
- **自動ログイン**：「ログイン状態を保持」チェックボックスで30日間有効なトークン発行
- **ログアウト**：明示的なログアウトボタン、トークン無効化

#### 2-1-3. パスワード管理
- **パスワードリセット**：
  - ログイン画面から「パスワードを忘れた」リンク
  - メールアドレス入力
  - リセット用リンクをメール送信（有効期限1時間）
  - 新しいパスワード設定画面へ遷移
- **パスワード変更**：
  - 設定画面から変更可能
  - 現在のパスワード入力必須
  - 新しいパスワード入力（確認用に2回）
  - 変更後、全セッション無効化（再ログイン必要）

#### 2-1-4. アカウント削除
- **削除フロー**：
  1. 設定画面の「アカウント削除」ボタン
  2. 確認ダイアログ（「本当に削除しますか？」）
  3. パスワード入力による本人確認
  4. 削除理由の選択（任意、改善のためのフィードバック）
  5. 最終確認（「30日以内なら復元可能です」の警告表示）
- **削除処理**：
  - 論理削除（deleted_atフラグ）
  - 30日間は復元可能
  - 30日経過後に物理削除（バッチ処理）
- **復元機能**：
  - 削除から30日以内にログイン試行
  - 「アカウントは削除待ちです。復元しますか？」と表示
  - 復元ボタンで即座に復元

### 2-2. オンボーディング

#### 2-2-1. 初回起動時のフロー
1. **ウェルカム画面**
   - アプリ名とロゴ表示
   - 「家計簿を楽しく続けよう！」のキャッチコピー
   - 「始める」ボタン

2. **アプリ説明（スライド3枚）**
   - スライド1：「支出を記録するだけでキャラが育つ」
   - スライド2：「記録を忘れるとキャラがしょんぼり」
   - スライド3：「予算を守ると特別な装飾品がもらえる」
   - 各スライドに視覚的なイラスト
   - スキップボタンあり

3. **最初のキャラクター獲得演出**
   - 光の球体が出現
   - 「カケボン・ノーマル」が登場
   - 「よろしくね！」のセリフ
   - 次へボタン

4. **支出入力デモ**
   - 「試しに支出を入力してみよう」のガイド
   - サンプル支出を入力させる（金額500円、カテゴリ食費など）
   - キャラクターが給餌アニメーション
   - 「これで経験値がたまるよ！」の説明

5. **予算設定（必須）**
   - 「今月の予算を設定しよう」
   - 金額入力（デフォルト100,000円）

   - スキップ不可

6. **完了画面**
   - 「準備完了！」
   - 「早速使ってみよう」ボタン
   - ダッシュボードへ遷移

#### 2-2-2. チュートリアルのスキップ
- スライド部分のみスキップ可能
- キャラクター獲得と予算設定はスキップ不可
- スキップした場合でも後から「ヘルプ」で確認可能

### 2-3. 支出入力機能

#### 2-3-1. 入力画面構成
- **画面上部（1/3）**：メインキャラクター表示
- **画面中部（1/3）**：金額入力エリア
  - 大きなテンキー（数字0〜9、クリア、確定）
  - 入力中の金額を大きく表示
  - カンマ区切り自動挿入（例：10000 → 10,000）
- **画面下部（1/3）**：カテゴリ選択エリア

#### 2-3-2. カテゴリ選択UI
- **表示形式**：プルダウンメニュー
- **デフォルト表示**：「カテゴリを選択」（プレースホルダー）
- **タップ時の動作**：
  - 画面下から上にスライドアップ（0.3秒、イージング）
  - 背景に半透明の黒オーバーレイ（opacity: 0.5）
  - メニューの高さは画面の60%
- **メニュー内容**：
  - 各カテゴリをアイコン＋テキストで表示
  - 最近使用したカテゴリが上部にハイライト表示
  - スクロール可能
- **選択時の動作**：
  - 選択したカテゴリが0.2秒で拡大→縮小のバウンスアニメーション
  - プルダウンが上にスライドして閉じる（0.3秒）
  - 選択されたカテゴリがボタン上に表示（アイコン＋名称）

#### 2-3-3. カテゴリ一覧（固定6個＋カスタム）
1. **食費**：🍴（オレンジ色）
2. **日用品**：🛒（グリーン）
3. **娯楽**：🎮（ピンク）
4. **交通費**：🚇（ブルー）
5. **光熱費**：💡（イエロー）
6. **その他**：📦（グレー）

- カスタムカテゴリ追加機能：設定画面から最大10個まで追加可能
- カスタムカテゴリは絵文字＋色を選択可能

#### 2-3-4. 日付・時刻設定
- **デフォルト**：現在日時が自動入力
- **変更方法**：金額入力画面の上部に「日時」ボタン
- **日時選択画面**：
  - カレンダーUI（過去30日間のみ選択可能）
  - 時刻選択（時・分の2つのドラムロール）
  - 未来日時は選択不可（グレーアウト）
  - キャンセル・決定ボタン
- **選択後の表示**：「◯月◯日 ◯時◯分の支出として記録」と小さく表示

#### 2-3-5. メモ機能
- **入力タイミング**：支出登録確定後
- **プロンプト表示**：「メモを追加しますか？」（3秒表示、自動で消える）
- **メモ入力画面**：
  - フルスクリーンのテキストエリア
  - プレースホルダー：「この支出について何かメモがあれば...（任意）」
  - 最大文字数：500文字
  - 文字カウンター表示（「245/500」形式）
  - 保存・スキップボタン
- **スキップ**：ボタンまたは画面外タップで即座にメイン画面に戻る

#### 2-3-6. バリデーションルール
- **金額**：
  - 必須
  - 1円以上、999,999円以下
  - 小数点入力不可（円単位のみ）
  - 0円は不可
- **カテゴリ**：必須
- **日時**：
  - 必須
  - 過去30日以内
  - 未来日時不可
- **メモ**：
  - 任意
  - 最大500文字

#### 2-3-7. 入力完了時の処理
1. バリデーションチェック
2. APIリクエスト送信（非同期）
3. 即座にキャラクター給餌アニメーション再生
4. API成功時：
   - 経験値獲得表示（「+10 EXP」とフェードイン）
   - レベルアップの場合は特別演出
5. API失敗時：
   - データをローカルストレージに一時保存
   - 「オフライン中です。後で同期します」トースト表示
   - 再試行ボタン表示

### 2-4. 履歴表示・編集機能

#### 2-4-1. 履歴一覧画面
- **表示形式**：無限スクロール方式
- **初期表示**：直近30日分
- **追加読み込み**：スクロール時に30件ずつ読み込み
- **グルーピング**：日付ごとにセクション分け
  - セクションヘッダー：「◯月◯日（曜日） 合計：◯,◯◯◯円」
  - その日の支出が時系列で並ぶ（新しい順）

#### 2-4-2. 履歴1行の表示内容
```
[カテゴリアイコン] [金額] [時刻]
                    [メモの冒頭20文字...] [メモアイコン]
```

- **左側**：カテゴリアイコン（40px × 40px、円形背景）
- **中央上段**：金額（大きく太字、「¥1,200」形式）
- **中央下段**：メモの冒頭20文字（薄いグレー、メモがある場合のみ）
- **右側上段**：時刻（「12:34」形式）
- **右側下段**：メモアイコン（メモがある場合のみ、📝）
- **背景**：タップ可能であることを示す微妙な影
- **区切り線**：各行の間に薄いグレーの線

#### 2-4-3. タップ時の動作
- **アニメーション**：タップした行が0.2秒で拡大（scale: 1.05）
- **遷移**：詳細画面へ遷移（右からスライドイン）

#### 2-4-4. 詳細画面の表示内容
```
[キャラクターアイコン]

金額：¥1,200（大きく中央表示）
カテゴリ：[アイコン] 食費
日時：2025年3月20日 12:34
メモ：コンビニでランチを購入...（全文表示）

登録日時：3月20日 15:45に登録（小さく薄く）

[編集] [削除]
```

#### 2-4-5. 編集機能
- **編集可能項目**：
  1. 金額
  2. カテゴリ
  3. メモ
  4. 日付
  5. 時刻
- **編集不可項目**：
  - 登録日時（システム記録として保持）
- **編集画面UI**：
  - 入力画面と同じレイアウト
  - 既存の値がプリセット表示
  - 変更したい項目のみタップして修正
- **保存処理**：
  - 保存ボタン押下で即座に反映
  - APIリクエスト送信
  - 成功時：「保存しました」トースト表示
  - 失敗時：「保存に失敗しました」エラー表示、再試行ボタン
- **キャンセル**：
  - キャンセルボタンで変更を破棄
  - 変更がある場合のみ確認ダイアログ表示

#### 2-4-6. 削除機能
- **削除ボタン位置**：詳細画面の最下部、赤色
- **確認ダイアログ**：
  - 「この支出を削除しますか？」
  - 「削除しても経験値は減りません」の注釈
  - キャンセル・削除ボタン
- **削除処理**：
  - 論理削除（is_deleted フラグを true に）
  - 削除データは「ゴミ箱」画面から30日間復元可能
- **削除後の動作**：
  - 履歴一覧に戻る
  - 「削除しました」トースト表示

#### 2-4-7. ゴミ箱機能
- **アクセス**：履歴画面の右上メニューから「ゴミ箱」
- **表示内容**：削除した支出の一覧（削除日時順）
- **復元機能**：
  - 各行に「復元」ボタン
  - タップで即座に復元
  - 復元後は通常の履歴に戻る
- **完全削除**：
  - 「ゴミ箱を空にする」ボタン
  - 確認ダイアログ後、全て物理削除
- **自動削除**：削除から30日経過したデータは自動的に物理削除（日次バッチ処理）

### 2-5. 予算管理機能

#### 2-5-1. 予算設定のタイミング
- **初回設定**：オンボーディング時に必須入力
- **月次更新**：
  - 毎月1日の初回起動時にプロンプト表示
  - 「今月の予算を設定しますか？」
  - 前月の予算額がデフォルト表示
  - 「前月と同じ」「変更する」「後で設定」ボタン
- **任意変更**：設定画面からいつでも変更可能

#### 2-5-2. 予算の設定単位
- **総予算のみ**：「月間◯◯,◯◯◯円」のように合計額のみ設定
- **カテゴリ別予算（オプション）**：
  - 「詳細設定」ボタンで展開
  - 各カテゴリごとに予算額を設定
  - カテゴリ別予算の合計が総予算を超える場合はエラー表示
  - 総予算のみ設定の場合、カテゴリ別予算は null

#### 2-5-3. 予算開始日
- **デフォルト**：毎月1日
- **カスタマイズ可能**：設定画面で1日〜28日から選択
- **用途例**：給料日が25日の人は25日〜翌月24日を1ヶ月とする

#### 2-5-4. 予算達成率の計算
```
達成率 = (当月の累計支出 ÷ 月間予算) × 100
```

- **累計支出の範囲**：
  - 予算開始日から現在日までの支出
  - 削除済み（is_deleted=true）の支出は除外
  - 未来日付の支出は除外

#### 2-5-5. 予算達成率の表示箇所
1. **ダッシュボード画面**：
   - プログレスバー（0〜100%は緑、100〜120%は黄色、120%〜は赤）
   - 数値表示（「予算の 75% を使用中」）
   - 残額表示（「残り ¥25,000」）

2. **キャラクター画面**：
   - 吹き出しで「予算の◯%を使ったよ」

3. **支出入力完了時**：
   - 一時表示「今日で予算の◯%」（3秒で消える）

#### 2-5-6. 警告表示の閾値
- **50%到達時**：
  - 通知：「予算の半分を使用しました」
  - ダッシュボードに「順調です」のメッセージ

- **80%到達時**：
  - 通知：「予算の80%に達しました。残り◯日で◯,◯◯◯円です」
  - キャラクターの表情が「心配そう」に変化
  - プログレスバーが黄色に変化
  - ダッシュボードに「注意」バッジ

- **100%到達時**：
  - 通知：「予算を超過しました」
  - キャラクターの表情が「困った」に変化
  - プログレスバーが赤色に変化
  - 超過額を明示（「予算を ¥5,000 超過しています」）

- **120%到達時**：
  - キャラクターが「浪費形態」に変化
  - 特別な警告演出（画面が赤くフラッシュ）

#### 2-5-7. 予算の変更フロー
- **即時適用 vs 翌月適用**：
  - 「今月から適用」「翌月から適用」を選択させる
  - 今月から適用の場合：
    - 「既存の支出データは変わりません。達成率が再計算されます」の警告
    - 確認ダイアログ表示
  - 翌月から適用の場合：
    - 「◯月◯日から新しい予算が適用されます」の確認表示

### 2-6. キャラクター育成システム

#### 2-6-1. 経験値獲得ルール

| 行動 | 獲得経験値 | 条件・補足 |
|------|-----------|-----------|
| 支出入力 | 10 | 1回あたり |
| 支出編集 | 5 | 入力の半分 |
| 支出削除 | 0 | ペナルティなし |
| 予算設定 | 50 | 新規設定または変更時 |
| 週次レポート確認 | 20 | 初回確認時のみ |
| 月次レポート確認 | 30 | 初回確認時のみ |
| 連続入力ボーナス | 50 | 7日連続で入力後、8日目 |
| 予算達成ボーナス | 100 | 月次レポート生成時 |

**特性による倍率**：
- カケボン・グルメが食費入力：10 × 1.5 = 15
- カケボン・ゴールドが全カテゴリ入力：10 × 1.2 = 12
- カケボン・レジェンドが全カテゴリ入力：10 × 2.0 = 20

**メインキャラクターボーナス**：
- メインキャラクターは常に 1.2倍

#### 2-6-2. レベルアップ必要経験値

```
次レベルまでの必要経験値 = 100 × (現在レベル)^1.5
```

| レベル | 累積必要経験値 | 次レベルまで |
|--------|---------------|-------------|
| 1→2    | 100           | 100         |
| 2→3    | 325           | 225         |
| 3→4    | 705           | 380         |
| 4→5    | 1,267         | 562         |
| 5→6    | 2,002         | 735         |
| 10→11  | 10,330        | 1,945       |
| 20→21  | 59,460        | 4,472       |
| 50→51  | 588,388       | 17,678      |

**レベル上限**：なし（無限に成長可能）

#### 2-6-3. 進化段階の定義

| 段階 | レベル範囲 | 名称 | 見た目の特徴 |
|------|-----------|------|-------------|
| 1    | 1〜4      | 幼体 | 小さく単純、手足なし |
| 2    | 5〜14     | 成長期 | 小さな手足が生える、色が鮮やか |
| 3    | 15〜29    | 成体 | 体型が個性的、尻尾の形が明確 |
| 4    | 30〜49    | 完全体 | 装飾が追加、模様や質感 |
| 5    | 50〜      | 究極体 | 光のエフェクト、オーラ |

**進化時の演出**：
1. 光に包まれる（2秒）
2. 体が変化（3秒）
3. ポーズを決める（1秒）
4. 「進化しました！」の大きな文字表示

#### 2-6-4. 形態変化システム

**判定タイミング**：
- 毎日午前0時（日次バッチ処理）
- 支出入力時（リアルタイム判定）

**判定ロジック**：
1. 直近7日間（当日含む）の全支出をカテゴリ別に合計
2. 最も支出額が大きいカテゴリを特定
3. そのカテゴリに対応する形態に変化

**形態一覧**：

| 形態名 | 発生条件 | 体色 | 尻尾の形 | 特徴 |
|--------|---------|------|---------|------|
| 通常形態 | 特定カテゴリが突出していない | クリーム色 | 雲型 | バランス型 |

## 3. 非機能要件

### 3-2. スケーラビリティ要件

**想定ユーザー数**：
- 初年度：1,000ユーザー
- 3年後：10,000ユーザー
- 5年後：50,000ユーザー

### 3-4. セキュリティ要件

#### 3-4-1. 認証・認可
- JWT使用、有効期限7日間
- リフレッシュトークン（30日間）
- パスワードはbcryptでハッシュ化（コスト係数12）
- セッション管理はRedis

#### 3-4-2. 通信セキュリティ
- 全通信をHTTPS必須（TLS 1.3）
- HSTS（HTTP Strict Transport Security）有効化
- CSP（Content Security Policy）設定

#### 3-4-3. データ保護
- 個人情報（メールアドレス）は暗号化して保存
- 支出データ自体は平文（クレカ番号等含まないため）
- データベースアクセスはVPN経由のみ

#### 3-4-4. 入力検証
- フロントエンド・バックエンド両方でバリデーション
- SQLインジェクション対策（ORMのパラメータバインディング）
- XSS対策（サニタイゼーション）
- CSRF対策（トークン使用）

#### 3-4-5. レート制限
- API呼び出し制限：1ユーザーあたり100リクエスト/分
- ログイン試行制限：5回失敗で15分ロック
- パスワードリセット：1時間に3回まで

12. `user_accessories`：ユーザー獲得装飾品

#### 4-4-2. 主要テーブル詳細

**users テーブル**
```sql
CREATE TABLE users (
    user_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email_verified BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP NULL
);

CREATE INDEX idx_users_email ON users(email);
```

**expenses テーブル**
```sql
CREATE TABLE expenses (
    expense_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    amount DECIMAL(10, 2) NOT NULL,
    category VARCHAR(50) NOT NULL,
    memo TEXT,
    recorded_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_deleted BOOLEAN DEFAULT FALSE,
    deleted_at TIMESTAMP NULL
);

CREATE INDEX idx_expenses_user_recorded ON expenses(user_id, recorded_at DESC);
CREATE INDEX idx_expenses_category ON expenses(user_id, category);
CREATE INDEX idx_expenses_deleted ON expenses(user_id, is_deleted);
```

**user_characters テーブル**
```sql
CREATE TABLE user_characters (
    user_character_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    character_master_id INT NOT NULL,
    nickname VARCHAR(50),
    level INT DEFAULT 1,
    experience INT DEFAULT 0,
    evolution_stage INT DEFAULT 1,
    health_status VARCHAR(20) DEFAULT 'healthy',
    last_fed_at TIMESTAMP,
    accessories JSONB DEFAULT '[]'::jsonb,
    total_fed_count INT DEFAULT 0,
    is_main BOOLEAN DEFAULT FALSE,
    acquired_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, character_master_id)
);

CREATE INDEX idx_user_characters_main ON user_characters(user_id, is_main);
```
